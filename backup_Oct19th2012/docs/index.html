User Interface

We complete the user interface by implementing the toggle show/hide feature of the options overlay. We will need to add some JavaScript code for this. Go ahead and open up the skeleton.js file (about time!) in the js folder in your favorite text editor. Locate the toggleOptions() function and observe it. The options variable stores the DIV element tagged by the id options_overlay. This is our overlay element. The visibility variable stores the current value of the display attribute of the overlay element. This attribute determines whether the element is hidden or not. Its possible values are the strings "none" (hidden), "inline" (visible) or "block" (visible). The overlay is obviously a block element since it will not be rendered in line with text, so we implement a toggle mechanism that sets the display attribute of options to "none" when it is "block" and "block" when it is "none". This can be accomplished using a simple if-else conditional structure. Recall that we bound click listeners to the map container (map_canvas). From now on, the options button acts as a toggle switch to show or hide the options overlay. Additionally, clicking anywhere else on the interface except for the toolbar will hide the overlay. This effectively emulates a toggle show/hide mechanism.
